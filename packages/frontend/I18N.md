# Internationalization (i18n) Guide

This project uses `react-i18next` for internationalization. The setup is optimized for Next.js App Router Client Components, allowing fast, localized UI without requiring heavy server-side middleware for locale-based routing.

## Infrastructure Setup

1. **`packages/frontend/src/i18n/config.ts`**: The core configuration file that initializes `i18next`. It automatically uses `i18next-browser-languagedetector` to detect the user's preferred language from their browser and persists their selection in `localStorage`.
2. **`packages/frontend/src/components/I18nProvider.tsx`**: A client-side provider that wraps the application. It ensures `i18next` is available via Context to all components and also handles dynamic `<html lang="...">` and `dir="..."` (for RTL support) updates when the language changes.
3. **Locales Directory (`packages/frontend/src/i18n/locales/`)**: Contains the JSON translation dictionaries (e.g., `en.json`, `es.json`).

## How to use translations in components

To translate text in a component, the component must be a **Client Component** (marked with `"use client"` at the top).

```tsx
"use client";

import { useTranslation } from "react-i18next";

export function MyComponent() {
  const { t } = useTranslation();

  return (
    <div>
      {/* Basic translation */}
      <h1>{t("myKey", "Default English Text")}</h1>

      {/* Translation with interpolation */}
      <p>{t("welcomeMessage", { username: "Alice" })}</p>
    </div>
  );
}
```

*Note: It is highly recommended to provide a default string as the second argument to `t()`. This acts as a fallback and makes the code more readable.*

## Adding a New Language

1. Create a new JSON file in `src/i18n/locales/` (e.g., `fr.json`).
2. Copy the structure from `en.json` and translate the values.
3. Open `src/i18n/config.ts` and import your new file:
   ```typescript
   import fr from "./locales/fr.json";
   ```
4. Add it to the `resources` object in `config.ts`:
   ```typescript
   resources: {
     en: { translation: en },
     es: { translation: es },
     fr: { translation: fr }
   }
   ```
5. Open `src/components/NavBar.tsx` and add an option to the language switcher dropdown:
   ```tsx
   <option value="fr" className="bg-[#080b14]">🇫🇷 FR</option>
   ```

## RTL (Right-to-Left) Support

The `I18nProvider` automatically sets `document.dir = i18next.dir()`.
If you add an Arabic (`ar`) or Hebrew (`he`) locale, `i18next` will automatically detect it as an RTL language and set `dir="rtl"` on the document.

You can style RTL-specific layouts in CSS/Tailwind using the `rtl:` prefix:
```tsx
<div className="ml-4 rtl:mr-4 rtl:ml-0">
  Content
</div>
```
